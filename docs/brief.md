# Project Brief

## Project Name
ChromaOS CRM

## Project Summary
ChromaOS CRM is the internal unified CRM and operational system for Chromapages and its sub-brands. The goal is to manage leads, companies, deals, tasks, activities, and AI-assisted workflows from one Firebase-based application.

This project is meant to become the internal system of record for multi-brand agency operations.

## Business Context
Chromapages is no longer just one brand with one sales flow. It is evolving into a parent brand with multiple vertical brands and service paths. That means the operating model needs to support:
- different offers
- different pipeline logic
- different brand contexts
- one owner/operator needing visibility across everything

Without a unified CRM, the business will default to scattered records, weak routing, missed tasks, and inconsistent next steps.

## Primary Goal
Create one internal Firebase-based CRM that manages:
- all leads
- all brand routing
- all pipeline stages
- all deals
- all tasks
- all activities
- all agent-assisted recommendations

## Secondary Goals
- standardize lead qualification
- speed up proposal prep
- reduce operational friction
- improve visibility across brands
- create a clean system for future team growth

## Core Problem
The current and future bottleneck is not offer definition. It is operational complexity. As more brands and offers come online, the business needs a structured system that answers:
- what came in?
- which brand does it belong to?
- what stage is it in?
- what needs to happen next?
- who owns the next step?
- what is it worth?

## Target Users

### Primary User
Eric / owner-operator who needs:
- one place to review all leads
- one place to track brand pipelines
- one place to manage next actions
- one place to use AI support in context

### Secondary Users
- future internal operators
- future sales/admin support
- OpenClaw agents working against structured CRM records

## Brands In Scope
- Chromapages
- BuiltExpert
- ServeStrategy

## Core Outcomes
The system should make it easier to:
- classify leads quickly
- route leads correctly
- manage tasks consistently
- track opportunities clearly
- use AI where it adds leverage
- avoid operational chaos

## Success Metrics
The CRM is successful if it:
- becomes the default internal lead/deal system
- reduces missed follow-up
- improves visibility across brands
- speeds up routing and proposal preparation
- gives the owner a fast operational dashboard

## Strategic Outcome
ChromaOS CRM should become the internal operating spine for Chromapages and all sub-brands, while giving OpenClaw agents a structured and persistent system of record to work from.\n
